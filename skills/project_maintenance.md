# Project Maintenance Skill

Use this rule for every AutoEE code, UI, workflow, demo-data, or content change:

- Update `README.md` in the same change.
- Update `README_START_HERE.txt` or module-specific docs when the run flow, architecture, capabilities, or boundaries change.
- Keep documentation aligned with the current project architecture, module responsibilities, frontend/backend data flow, fake-data outputs, not-connected capabilities, and local run instructions.
- Treat documentation as part of implementation quality, not a follow-up task.

This is a persistent project convention for AutoEE collaboration.
