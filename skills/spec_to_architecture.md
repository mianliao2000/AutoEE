# Skill: Spec to Architecture

Use when converting a product requirement into an initial power-stage topology
and constraint matrix.

Inputs:
- `specs/current_spec.yaml`
- User constraints and open questions

Steps:
- Check required electrical, thermal, control, protection, and testing fields.
- List missing or ambiguous requirements.
- Propose topology candidates and tradeoffs.
- Record assumptions explicitly.

Checks:
- Units are explicit.
- Operating points are defined.
- Human review is required before freezing architecture.
